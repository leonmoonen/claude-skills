# SFF-VI Phase 2 Proposal Review (Norwegian Centers of Excellence)

Help a research group get their SFF-VI Phase 2 application onto RCN's final funded SFF list. 
This is a peer-review-style critique of a 10-year, multi-group center proposal. 
It is considerably bigger in scope than a single-PI grant, and assessed by a genuinely different mechanism, 
so don't assume the FRIPRO playbook (see `fripro-proposal-review`) transfers directly, even though the four criteria names look similar.

## The one fact that should shape every review

SFF-VI Phase 2 is decided by **comparative ranking, not a qualification threshold**. 
Each application's referee panel only ever reviews *that one application* in isolation and cannot compare it to others. 
The real decision is made afterward by a Scientific Committee that reads every Phase 2 application side by side and ranks all of them, recommending roughly 12 (out of the 36 that advanced from Phase 1, spread evenly across the three subcommittees/domains) for funding, with some guaranteed slots per subject-area domain and the remainder filled by merit regardless of domain (check the live call page for the exact current split). 
This means the review should never treat "does this clear a bar" as the operative question: it should treat "would a committee reading this next to ~30 other center-scale proposals remember it, believe it, and rank it near the top" as the operative question. 
Read `references/assessment-criteria.md` for the full text of each criterion (including the Phase-2-only additions) plus guidance on what tends to separate a proposal that ranks well from one that doesn't. Read this before writing detailed feedback.

`references/sff-call-details.md` has eligibility, funding scale, attachment/page-limit, budget, process, and organizational (governance/SAC/co-location) facts specific to this call. 
Read it for questions on those topics or when giving advice that touches them. 
`references/drafting-tips-qa.md` has a checklist of concrete, easy-to-miss compliance and drafting pitfalls straight from RCN's own applicant Q&A (formatting rules, CV template dos/don'ts, partner/budget quirks) — worth a pass in Step 5 below.

If anything in the reference files seems to conflict with what the user tells you about the live call page (this is a rare, once-per-generation call, and rules have shifted between SFF generations and even between Phase 1 and Phase 2 of the same one), trust the user's more current information and say so.

## Inputs to expect

- **Project description** (draft: 1 front page + max 15 pages + 1 reference page) — the main document.
- **CV + track record** for the centre director (2+2 pages) and each group leader (2+2 pages, 2–6 people).
- **10-year budget** (basic funding + supplementary funding, Excel template).
- Sometimes a **draft rebuttal** responding to a referee panel's preliminary assessment. 
  This is a distinct, later-stage document (see Step 6) and should be reviewed differently from the original proposal.
- Sometimes just questions about eligibility, process, or strategy with no draft yet. 
  Answer from the reference files rather than forcing the full review workflow.

Since this is Phase 2, confirm early (if not already clear) that the application was actually invited forward from Phase 1, that the centre director and named group leaders are unchanged from Phase 1, and that the project being described is recognizably the same project as Phase 1 (just more developed).
RCN will not read the Phase 1 text unless there's doubt about this, so a Phase 2 draft that has drifted substantially from the Phase 1 pitch is a real risk worth naming explicitly, not just a stylistic choice.

## Review workflow

### Step 1: Read the material

Read the full project description before forming opinions — a first pass for the big picture (the center's core scientific bet and organizational design), a second for detail. 
Read the center director's CV/track record and each group leader's, checking that claims in the project description about the team's expertise are actually supported by the CVs.
If a budget is included, check it against the research themes and team size described.

If this is a PDF, use the `pages` parameter for the full 17-page document. 
Note which assessment criteria each part of the document is answering.
Phase 2 proposals are expected to explicitly address all four criteria somewhere in the text (per RCN's own guidance to "clearly address the assessment criteria"), so a document that never visibly engages with, say, Impact or the governance sub-points of Implementation has a real gap, not just an emphasis choice.

### Step 2: Structured summary

Before critiquing, show that you understood the proposal. Cover:

1. **Central scientific bet** — the ambitious, beyond-state-of-the-art claim, in your own words.
2. **Why a center** — what specifically requires a 10-year, multi-group structure rather than a smaller project (or: is this case even made explicitly?).
3. **Team and organization** — center director, group leaders (and their apparent complementary roles), partners, governance.
4. **Requested scope** — total ask against the NOK 80–160M/10-year envelope, basic vs. supplementary funding balance, any own-financing signal.
5. **Domain** — which of the three subject areas (Humanities & Social Sciences / Life Sciences / Natural  Sciences & Technology) this is competing within, and whether it's framed as interdisciplinary.

### Step 3: Criterion-by-criterion assessment

Work through all four criteria from `references/assessment-criteria.md`, including the Phase-2-only sub-points. For each:

- Assess strength in *relative* terms; since there's no absolute pass mark, phrase this as "how this would read next to other center-scale proposals in the same domain" rather than a 1–7 score.
- Explicitly check the Phase-2-only additions (ethics/safety/gender dimension under R&D quality; talent attraction and communication under Impact; group complementarity, governance, physical organization, and gender-imbalance planning under Implementation) — these are new relative to Phase 1 and easy to under-specify if the draft evolved from a Phase 1 text rather than being rebuilt around the fuller criteria.
- Flag anywhere the "why does this need center-scale, multi-group organization" case isn't made. 
  This question cuts across all four criteria (state-of-the-art ambition, R&D design, impact on the environment, and "goes beyond what individual group leaders could achieve alone" under Implementation) and is the single most common soft spot in center proposals that read as an aggregation of individually strong PIs.

### Step 4: Team and governance consistency check

Specific to center-scale proposals, verify that: 
- each named group leader's stated expertise maps to a distinct, identified piece of the centre's research design (no unexplained redundancy, no critical method with no owner), 
- at least one group leader beyond the director clearly has a track record of significant research achievements (an explicit sub-criterion), 
- a governance (board, Scientific Advisory Committee, decision-making) is concretely described with named roles rather than asserted in the abstract, and 
- co-location / physical organization is addressed, given it's now an explicit Phase 2 sub-criterion. 
If the leadership group (director + group leaders) is gender-imbalanced, check for a concrete talent-development plan for the underrepresented gender; this is directly and explicitly scored when an imbalance exists.

### Step 5: Compliance and drafting pitfalls

Run through `references/drafting-tips-qa.md` as a checklist: 
- template/page-limit compliance (content beyond limits or outside templates is simply not assessed, not just penalized), 
- CV/track record formatting, 
- partner-vs-individual-collaborator confusion, 
- budget/fellowship quirks, and 
- whether the Impact criterion is actually answered in the project description text itself (not left to application-form fields the committee won't necessarily see).

### Step 6: Rebuttal review (only if a rebuttal draft is provided)

If reviewing a rebuttal rather than the original proposal: remember RCN's rebuttal mechanism exists only to correct **scientific misunderstandings** in the referee panel's preliminary assessment, not to re-litigate disagreements about judgment, priorities, or scoring. 
Check the rebuttal stays within that scope: a rebuttal that reads as arguing the panel was simply wrong to weigh something the way it did is likely to be disregarded; one that politely and specifically shows where the panel misread a claim (with a pointer to exactly where the original text already said otherwise) is much more likely to be takenup.

### Step 7: Feedback

Structure output using the template below. 
Be specific: 
— cite the page/section where a claim appears or is missing, and 
— be constructive. 
Distinguish "likely present but not surfaced" gaps from genuine substance gaps, especially if the draft is visibly incomplete or still close to Phase-1 length, rather than using the fuller Phase 2 page budget.

If the draft is clearly partial (an outline, a single section, much shorter than the ~17-page limit), say so up front and caveat throughout that unaddressed points may simply not have made it into this excerpt, rather than being genuinely absent from the full document; don't let a short draft read as a harsher verdict than it deserves.
